# CoCRM v5 注册逻辑详细说明

## 1. 项目概述

本项目 CoCRM v5 是一个客户关系管理系统，实现了完整的用户注册、登录和身份验证功能。系统支持两种类型的用户：
- 代理商 (Agent) - ID类型以1开头
- 客户 (Customer) - ID类型以6开头

## 2. 注册流程

### 2.1 前端流程
1. 用户访问注册页面
2. 填写注册表单（邮箱、密码、姓名、电话、国家、性别、账户类型等）
3. 如果是客户类型，需要提供有效的邀请人ID
4. 系统实时验证邮箱唯一性
5. 提交注册请求

### 2.2 后端流程
1. 验证输入数据格式
2. 检查邮箱唯一性
3. 验证邀请人（如果适用）
4. 生成唯一用户ID
5. 创建用户登录记录
6. 创建用户信息记录
7. 建立推荐关系（如果有邀请人）

## 3. 核心组件

### 3.1 控制器
- `Front/AuthController@register` - 处理注册请求

### 3.2 服务类
- `UserRegistrationService` - 处理注册业务逻辑

### 3.3 模型
- `UserLogin` - 存储用户登录信息
- `UserInfo` - 存储用户详细信息
- `IdSequence` - 管理用户ID序列

## 4. 数据验证

### 4.1 输入验证规则
- 邮箱：必需，有效邮箱格式，最大255字符
- 密码：必需，最小6字符，需确认
- 姓名：必需，最大100字符
- 电话：可选，最大20字符
- 账户类型：必需，1(代理)或2(客户)
- 邀请人ID：客户类型必需，有效用户ID

### 4.2 业务验证规则
- 邮箱唯一性检查
- 邀请人有效性检查（存在且启用）
- 代理商可以没有邀请人，客户必需有邀请人

## 5. ID生成策略

系统采用序列号方式生成用户ID：
- 代理商：从1001开始递增
- 客户：从600001开始递增

## 6. 推荐关系

系统支持多层级推荐关系，每个用户可以有上级推荐人，形成用户架构树。

## 7. 安全措施

- 密码使用Hash加密存储
- 邮箱唯一性验证
- 邀请人有效性验证
- 输入数据验证和清理
- 使用事务保证数据一致性
