# CoCRM v5 注册逻辑设计

## 1. 旧项目注册逻辑概述

从旧项目 CoGMTK CRM v3 中提取的关键注册逻辑包括：

### 1.1 用户注册流程
- 邮箱验证
- 邀请人验证
- 账户类型选择（代理商/客户）
- 个人信息收集（姓名、电话、国家等）
- 性别选择
- 密码确认

### 1.2 验证规则
- 邮箱唯一性验证
- 邀请人有效性验证（仅对客户类型）
- 密码强度验证（至少6位）
- 必填字段验证

### 1.3 数据库交互
- 创建用户记录
- 创建用户登录记录
- 创建用户信息记录
- 分配初始角色
- 记录推荐关系

## 2. 新项目注册逻辑实现

### 2.1 前端实现
- 注册页面表单设计
- 实时验证反馈
- 邀请人验证接口调用
- 注册提交接口调用

### 2.2 后端实现
- 注册控制器逻辑
- 用户验证服务
- 推荐关系建立
- 角色分配逻辑

## 3. 邀请人机制
- 邀请人ID必填验证（针对客户类型）
- 邀请人状态验证（启用/禁用）
- 多层级推荐关系维护

## 4. 账户类型
- 代理商（Agent）：可以发展下级代理和客户
- 客户（Customer）：只能作为终端用户

## 5. 安全措施
- 邮箱防重复注册
- 邀请链验证
- 输入数据过滤
- 密码加密存储
