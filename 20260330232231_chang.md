## 3. Authentication System / 认证系统

### 3.1 JWT Authentication (非Session)
- **`app/Services/JwtService.php`**: Custom JWT service using firebase/php-jwt
  - Token generation with SSO support (stores JTI in cache)
  - Token parsing, validation, blacklisting
  - Token refresh within configurable window
  - Custom salt for enhanced security

### 3.2 Single Sign-On (SSO) / 单点登录
- **`app/Http/Middleware/SingleSignOn.php`**: Enforces one active session per user
  - Cache key: `sso:{guard}:{user_id}` → stores active JTI
  - New login invalidates previous session
  - Returns code 4003 for SSO conflict

### 3.3 Auth Guards / 认证守卫
- **`config/auth.php`**: Configured `admin` and `user` guards with eloquent providers
- **Admin guard** → `App\Models\Admin`
- **User guard** → `App\Models\UserLogin`

## 9. Core Business Logic / 核心业务逻辑

### 9.1 User Registration (from old RegisterController + RegisterEnMiddleware)
- **`app/Services/UserRegistrationService.php`**:
  - Generates unique user_id from id_sequences (agent from 1001, customer from 600001)
  - Builds family_tree: comma-separated ancestor chain (e.g., "1001,1002,1003,600001")
  - Determines group/commission from parent agent's settings
  - Handles ECN/STP group logic
  - Validates blacklist, email uniqueness, phone uniqueness, inviter validity
  - Creates user_login, user_info, user_auth records
  - Updates agent_descendants closure table

### 9.2 Agent Hierarchy (family_tree)
- **`app/Services/FamilyTreeService.php`**:
  - Manages agent→sub-agent→customer hierarchy
  - family_tree format: "1001,1002,1003,1004,600001" (comma-separated)
  - Example: Agent 1006's customer 600001 → family_tree = "1001,1002,1003,1004,1005,1006,600001"
  - Rebuild methods for tree and descendants table
  - Statistics: direct/indirect agents, direct/indirect customers
  - Network tree structure for display

### 9.3 Commission/Rebate System (from old PositionSummaryController)
- **`app/Services/CommissionService.php`**:
  - Real-time commission calculation based on open positions
  - Formula: volume_lots * spread_value * spread_ratio * (agent_rate - user_rate)
  - Settlement for closed positions within date range
  - Commission record creation and settlement workflow

### 9.4 Position Summary (from old positionSummarySearch, subPositionSummarySearch)
- **`app/Http/Controllers/Front/PositionSummaryController.php`**:
  - Overview: position summary for agent's network
  - search(): Detailed position search with date range, symbol filter
  - subSearch(): Sub-agent network position summary
  - clickSearch(): Individual user trade detail view

## 15. Key Architecture Decisions / 关键架构决策

1. **JWT over Session**: All API auth uses JWT tokens, not sessions (requirement #31)
2. **SSO**: Single active session per user enforced via cache-based JTI tracking (requirement #27)
3. **Button-level RBAC**: Permissions support menu/page/button types with separate admin/front guards (requirement #23, #32)
4. **Dual UI**: Two complete UI frameworks (naui + layui) with dynamic switching (requirement #4, #19, #34)
5. **Multi-language**: Full i18n support for backend (Laravel lang files) and frontend (JS lang files) (requirement #13, #24, #28)
6. **Family tree**: Comma-separated ancestor chain for O(1) ancestor lookup (requirement #7-9)
7. **Agent descendants closure table**: Pre-computed for efficient hierarchy queries (requirement #12)
8. **Separate ID sequences**: Agents from 1001, customers from 600001 (requirement #5-6)
